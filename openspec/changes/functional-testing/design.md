## Context

代码已开发完毕，需要进行首次功能验证。测试环境通过 docker-compose 启动，包含 MySQL、Go 后端、Nginx 前端三个容器。

## Goals / Non-Goals

**Goals:**
- 验证 docker-compose 能一键启动全部服务
- 验证 6 个后端 API 接口功能正确
- 验证 DeepSeek AI 查词链路端到端可用
- 验证前端页面渲染和交互正常
- 修复发现的阻塞性 bug

**Non-Goals:**
- 不写自动化测试代码（后续再做）
- 不测试通义千问（无 Key）
- 不做性能/压力测试

## Decisions

### 测试策略：API 优先 + 浏览器验证

先用 curl 直接调用 API 验证后端逻辑，确保数据链路畅通后再通过浏览器验证前端渲染。这样能快速定位问题是前端还是后端。

### 测试顺序

按业务流程依赖关系排序：
1. 服务启动 → 2. 注册 → 3. 登录 → 4. AI 查词 → 5. 保存 → 6. 列表 → 7. 删除 → 8. 前端浏览器验证

## Risks / Trade-offs

- **Docker 构建耗时**: 首次构建可能需要较长时间下载依赖
- **AI API 调用频率**: 测试查词时注意不要频繁调用，避免触发限流
- **MySQL 启动延迟**: 后端有重试逻辑，但需观察实际连接情况
